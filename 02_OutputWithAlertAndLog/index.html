What's the difference between alert() and console.log()?

  • alert() creates an alert message in a dialog box.
  • console.log() displays data on the browser's Javascript console.

What's one situation where you might use an `alert()` to tell the user something?

You might warn a user that the link they have clicked will navigate them away from the page they are on without saving some changes they have made.

You could also warn a user if they are about to do something that may have unintended consequences, like deleting something from a database that will delete other data.

Or, you could alert a user that if they navigate away from a page they will miss out on the spectacular deals and bargains available to them on that page!!! Only $19.99!!

What's one way that `alert()` might get obnoxious?

You have to click okay in order to proceed, so it halts the delivery of the information you have requested.

Would it make sense to use `console.log()` to tell the user things? Why or why not?

No. Users generally won't know how to open the Javascript console. And it certainly isn't something that users should be expected to do in order to interact with a web page.

What's one way that `console.log()` might be a good tool for you to use?

You could add notes to yourself as you code, possibly as a debugging tool of sorts.
